# Инструкция по деплою на Timeweb Cloud

Для развертывания проекта на Timeweb Cloud рекомендуется использовать раздел **"Apps"** (Приложения). Это позволит автоматически собирать проект из GitHub и обеспечит стабильную работу SSR (Server Side Rendering).

## 1. Подготовка репозитория
1. Убедитесь, что все изменения закоммичены и отправлены в ваш GitHub репозиторий.
2. Файлы `Dockerfile` и `.dockerignore` уже добавлены в проект.

## 2. Настройка в панели Timeweb Cloud
1. Зайдите в панель [Timeweb Cloud](https://timeweb.cloud/).
2. Выберите раздел **"Apps"** (Приложения) -> **"Создать"**.
3. Выберите источник: **GitHub**.
4. Авторизуйтесь и выберите ваш репозиторий `cheboko`.
5. В настройках сборки выберите **"Docker-контейнер"**.
6. Путь к Dockerfile: `./Dockerfile`.

## 3. Переменные окружения (Environment Variables)
В разделе **"Переменные окружения"** добавьте данные из вашего `.env.local`:
- `NEXTAUTH_SECRET` (произвольная секретная строка)
- `NEXTAUTH_URL` (ваш домен на Timeweb, например `https://app-123.timeweb.cloud`)
- `NEXT_PUBLIC_SUPABASE_URL` (URL вашего проекта Supabase)
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` (Анонимный ключ Supabase)
- `SUPABASE_SERVICE_ROLE_KEY` (Секретный ключ, если используется на бэкенде)
- `DATABASE_URL` (Строка подключения к БД)

## 4. Запуск
1. Нажмите **"Деплой"**.
2. Timeweb Cloud сам скачает код, соберет Docker-образ и запустит приложение на порту 3000 (внутреннем).
3. В настройках домена можно будет привязать свой домен или использовать бесплатный технический.

---
> [!NOTE]
> Использование Docker гарантирует, что приложение будет работать в той же среде, что и при локальной проверке, исключая проблемы с версиями Node.js на хостинге.
